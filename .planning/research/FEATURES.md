# Research: Features

**Project:** IAM Website Repositioning
**Domain:** B2B product-marketing website refresh
**Researched:** 2026-04-01
**Confidence:** MEDIUM

## Table Stakes

- Clear homepage explanation of what the company sells now
- Accurate product names and package structure
- Consistent value proposition across homepage, pricing, and product pages
- Working contact CTAs and navigation on all edited routes
- Bilingual parity for commercially important pages
- Credible product imagery that does not undermine trust

## Differentiators

- Stronger product framing around `IAM mobiel` instead of generic `2-in-1` repetition
- Positioning that explains single-use, dual-use, and premium branded variants clearly
- Terminology that better fits the product story, for example `programs` instead of blanket `games`
- Cleaner public-facing claims around ownership model, for example `no license costs` rather than vague `free updates`

## Anti-Features

- Large redesign effort that delays the messaging correction
- New platform or CMS work during a focused content refresh
- Leaving old package blocks or old counts live in secondary pages
- Updating only Dutch or only English variants

## Complexity Notes

- Homepage updates are straightforward but visible
- `IAM mobiel` package restructuring is moderate complexity because it touches product naming, copy, pricing, and comparisons
- Removing outdated sections is moderate complexity because duplicate blocks exist in multiple files
- Image replacement or retouching depends on available approved assets

## Dependencies

- Package naming and positioning must be decided before product-page cleanup is finished
- Affected routes must be inventoried before text replacement to avoid stale claims
- English variants depend on Dutch source updates plus manual review

## Roadmap Implications

- Brand and package framing should be handled before deep page cleanup
- Bilingual sync should be a dedicated release concern, not an afterthought
