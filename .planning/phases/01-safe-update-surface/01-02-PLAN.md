Per D-04: The script MUST be reusable (re-runnable after each phase). Per D-05: Target these exact phrases. Per D-06: Group output by file family, not by phrase.

**Script requirements:**

1. **Shebang and usage:** `#!/usr/bin/env bash` with usage help via `--help` flag. Accept `--summary` (counts only) and `--full` (line-by-line hits) modes. Default to `--summary`.

2. **Target phrase patterns (per D-05):**
   - `games` (case-insensitive, but flag compound words like "sportgames" with a note)
   - `free updates` (case-insensitive)
   - `gratis updates` (case-insensitive)
   - `100+` (literal match)
   - `Choose Your Package` (case-insensitive)
   - `Kies Uw Pakket` (case-insensitive)
   - `Kies Jouw Pakket` (case-insensitive)
   - `2-in-1` (literal match)
   - `2 in 1` (literal match)

3. **File family grouping (per D-06):** Read the inventory.md file to get family definitions. For each family, group all hits from the shell page, NL partial, and EN partial together under a single `=== Family: {name} ===` header. If a simpler approach is needed, hardcode the 18 known families (they are stable -- the site structure does not change dynamically).

4. **Output format for `--full` mode:**
   ```
   === Family: index ===
     index.html:
       L42: "100+ games" [games, 100+]
       L180: "Choose Your Package" [Choose Your Package]
     partials/index-nl.html:
       L38: "100+ games" [games, 100+]
     partials/index-en.html:
       L35: "100+ games" [games, 100+]
     Total: 4 hits

   === Family: mobiele-vloer ===
     ...
   ```

5. **Output format for `--summary` mode:**
   ```
   Family                   | Shell | NL    | EN    | Total
   -------------------------+-------+-------+-------+------
   index                    |   2   |   1   |   1   |   4
   mobiele-vloer            |   0   |   3   |   3   |   6
   ...
   -------------------------+-------+-------+-------+------
   TOTAL                    |   X   |   Y   |   Z   |  ALL
   ```

6. **Working directory:** The script must be run from the repo root (not from the tools directory). Add a check at the top: if `index.html` does not exist in the current directory, print an error and exit.

7. **Exclude `.planning/` directory** from all grep operations.

8. **Exit code:** Exit 0 if the script runs successfully (even if phrases are found -- finding phrases is expected and informational).

Make the script executable with `chmod +x`.
